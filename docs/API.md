# API 接口文档

本项目提供少量 HTTP API，用于训练新模型后清理旧的系统推荐数据、并用当前模型重新生成推荐。

> 下文所有接口的 `host` 默认为 `localhost:8000`，与 Web 页面共用同一服务。

---

## 背景：为什么要清理推荐

系统推荐记录在数据库中标记为 `rate_type = SYSTEM_RATE`（系统推荐），与用户手动打标数据 `rate_type = USER_RATE`（喜欢 / 不喜欢）相互独立。

当你重新训练了模型后，**旧的系统推荐结果并不会自动更新**，它们仍由上一次的模型生成。此时需要手动清理旧推荐，再用新模型重新生成，推荐列表才会反映最新模型的判断。

本文档的两个接口**只操作 `SYSTEM_RATE` 数据，不会影响你的用户打标数据**。

| 字段 | 值 | 说明 |
|------|----|------|
| `USER_RATE` | 1 | 用户手动打标（喜欢/不喜欢），清理接口**不会**动这部分 |
| `SYSTEM_RATE` | 2 | 系统根据模型生成的推荐，清理接口只针对这部分 |

---

## 1. 清理旧推荐

删除所有系统推荐记录（`rate_type = SYSTEM_RATE`），保留用户打标数据。清理后推荐列表会变空，直到触发下一次重新推荐或定时任务。

### 请求

```
POST /api/clear-recommend
```

无请求参数，无请求体。

### 响应

成功（200）：

```json
{
  "success": true,
  "deleted": 128
}
```

失败：

```json
{
  "success": false,
  "error": "错误信息"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `success` | bool | 是否成功 |
| `deleted` | int | 实际删除的记录数（仅成功时返回） |
| `error` | string | 失败原因（仅失败时返回） |

### 调用示例

```bash
curl -X POST http://localhost:8000/api/clear-recommend
```

---

## 2. 重新推荐

清理旧推荐后，立即用**当前已训练的模型**对全部数据重新预测并生成推荐。相当于「清理旧推荐 + 立即推荐」的组合操作。

### 请求

```
POST /api/re-recommend
```

无请求参数，无请求体。

### 前置条件

- 必须已训练过模型（`data/model/model.pkl` 存在），否则推荐阶段会失败。
- 建议先有足够的用户打标数据（≥ 300 条）以保证模型效果。

### 响应

成功（200）：

```json
{
  "success": true,
  "deleted": 128,
  "total": 1500,
  "recommended": 96
}
```

无数据可供推荐时：

```json
{
  "success": true,
  "deleted": 128,
  "total": 0,
  "recommended": 0,
  "warning": "没有可供推荐的数据"
}
```

失败：

```json
{
  "success": false,
  "error": "推荐失败: ...",
  "deleted": 128
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `success` | bool | 是否成功 |
| `deleted` | int | 清理掉的旧推荐记录数 |
| `total` | int | 本次参与预测的数据总数 |
| `recommended` | int | 本次被模型判定为「喜欢」并写入推荐的数量 |
| `warning` | string | 无数据时的提示（仅此情况出现） |
| `error` | string | 失败原因（仅失败时返回） |

> 注：即使推荐阶段失败，`deleted` 仍会返回清理阶段已删除的数量，便于排查。

### 调用示例

```bash
curl -X POST http://localhost:8000/api/re-recommend
```

---

## 页面入口

除直接调用 API 外，也可在 **模型管理页面**（`/model`）使用图形按钮：

| 按钮 | 对应接口 | 说明 |
|------|----------|------|
| 清理旧推荐 | `POST /api/clear-recommend` | 仅清理，需手动等待下次定时推荐 |
| 重新推荐 | `POST /api/re-recommend` | 清理 + 用当前模型立即重新推荐 |

点击按钮会弹出二次确认框，操作结果会显示在按钮下方。

---

## 典型使用流程

1. 在 `/tagit` 页面打标积累数据，或导入已有打标数据库
2. 在 `/model` 页面点击「开始训练」训练新模型（或调用 `/do-training`）
3. **训练完成后**，点击「重新推荐」（或 `POST /api/re-recommend`），用新模型刷新推荐列表
4. 回到推荐页面对结果进行确认，确认后的数据转为打标数据，用于下一轮训练

> 如果只想清空推荐、暂不重新生成（例如模型还没训好），用「清理旧推荐」即可。
